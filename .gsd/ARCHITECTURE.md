## Data Flow

1. **User Action:** User interacts with UI components (e.g., clicks "Add to Cart").
2. **Dispatch:** Component dispatches a Redux action (sync or async thunk).
3. **Store Update:** Reducers/ExtraReducers update the Redux state.
4. **UI Re-render:** Components subscribed to the state via `useAppSelector` re-render.
5. **Side Effects:** Async thunks handle API calls via Axios and persist state if needed.
